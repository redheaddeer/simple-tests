Если требуется запускать тесты из jar файла (невозможен запуск через gradle), то необходимо запускать Cucumber "вручную".
Для этого считываем параметры запуска (`java -Dcucumber.options="--tag @rest" -jar test.jar`), ищем параметр
'cucumber.options' и парсим в параметры запуска cucumber.Main.run().

При таком запуске игнорируются аннотации @Before / @After, 
поэтому если требуется выполнять собственные пред-/постусловия, то вызывать необходимо именно run() метод, 
указывая вызов методов before()/after() непосредственно до/после вызова запуска cucumber. 
Если архитектура тестов позволяет обойтись без пред-/постусловий, то можно вызывать cucumber.Main.main() - 
метод main() содержит вызов метода run() и самостоятельно завершает работу всего приложения
(внутри main() присутствует System.exit() с кодом выполнения run()).


Выбрали запуск main() - можем вызвать before() вручную, after() не сработает - программа завершится раньше. 
Выбрали запуск run() - можем вызвать before() и after() вручную, System.exit() вызываем сами. 
Аннотации @Before/@After не работают в обоих случаях. 

Данный модуль написан на Kotlin версии 2.1.20, версия Java - 17.
Для тестов REST API используется библиотека io.restassured версии 5.5.0, для проверок асинхронного API 
(менеджер очередей Kafka) - библиотеки spring-kafka и spring-kafka-test пакета org.springframework.kafka версии 2.8.11.
Тесты UI реализованы с использованием Selenium 4.14.1.
Для управления сценариями - библиотека io.cucumber версии 7.15.0.
